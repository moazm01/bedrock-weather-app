The Global Problem & Bedrock's Resilience

02 / 11

🌍 Global Environmental Threats

Extreme weather anomalies, destructive seismic activities, and sudden localized hazard blocks (such as fires, floods, landslides, or road collapses) represent a universal danger to travelers worldwide. When severe events hit, electrical power grids and cellular network towers are usually the first systems to collapse.

🔋 Power Constraint Solution: Bedrock features a high-contrast theme that scales to dark backgrounds to preserve battery lifetime under blackout conditions.

📡 3-Tier Offline-First Sync Architecture

Standard navigation tools crash completely in signal-denied zones. Bedrock implements a structured three-level pipeline fallback to safeguard user safety:

Tier 1: Live Server Firestore Synchronization (Online Stream)
Tier 2: SharedPreferences & SQLite Local Caches (Offline Data)
Tier 3: Immutable Static Default Assets (Initial Configuration)

Ensures warnings remain accessible even when satellite connections drop.

Metaphors & Graphics

Unified HUD Cockpit & Vector Rendering

03 / 11

HUD Cockpit Interface Metaphor

Rather than nested settings pages, the app operates as a **tactical HUD command console**. A custom map provides the base layer, while critical weather graphs and alert windows float on top, matching the mental model of cockpit operations.

Pulsing Threat Circles (Graphics Animation): Concentric halos pulse on the canvas driven by tickers to warn users of hazard boundaries.

🎨 Custom Graphics Canvas Engine

Downloading raster maps uses megabytes of data. Bedrock draws maps, forecasts, and trajectories locally using vector paths:

  • Vector Maps: Custom coordinate scaling and path translation (`zoomScale`) drawn on canvas.
  • Spline Charts: Connects temperature data points using smooth bezier math curves.
  • Solar Trajectories: Solves trigonometry equations on canvas: `centerX + radius * cos(angle)`.

HCI Principles

Good Design Principles Applied

04 / 11

👁️ Visibility of System Status

Heuristic 1

Ensures users are informed in real-time. Includes state-loading spinners, progress indicators on forms, and live system-wide alert banners at the top of the HUD map shell.

🗺️ Natural Mapping & Affordance

Heuristic 2

Interactive widgets use physical cues (buttons scale down on tap). Menus map database status options directly to emojis (e.g. ⛰️ landslide, 🌊 flood) to align with users' real-world models.

🚫 Error-Preventative Constraints

Heuristic 5

Restricts invalid inputs through validator syntax. Forms disable buttons during active updates to prevent duplicate writes, and require confirmation dialogs before sending data.

🔄 Consistency & Standards

Heuristic 4

Adheres strictly to standard Material Design rules. Integrates adaptive platform switches (`Switch.adaptive`) that automatically adjust to native iOS or Android layouts.

Accessibility

Universal Design & WCAG 2.1 Compliance

05 / 11

🌓 Visual Contrast Standard (1.4.3)

The interface contrast ratio exceeds **7.0:1** (above the WCAG AA minimum of 4.5:1). High visual contrast is a critical design choice that ensures screen readability for visually impaired users and under direct sunlight.

Legibility Gate: Light gray text elements styled against pure white backgrounds maximize display clarity in well-lit presentation spaces.

♿ Assistive Technologies & Targets

Ensuring the app remains accessible to users regardless of physical ability:

  • Fitts's Law Touch Targets: Controls are scaled to a minimum of **50 logical pixels** (exceeding standard 44px) to prevent mis-clicks.
  • Spacing Grids: Standardized 4dp layout margins separate adjacent inputs.
  • Screen Reader Support: Descriptive labels, explicit tooltips, and wrapped `Semantics` tags.

User Experience

Minimizing Cognitive Load & Gamification

06 / 11

🧠 Recognition Rather than Recall

In emergency situations, cognitive capabilities are strained. Bedrock lists key contact numbers (Rescue, Hospital networks) directly in a sliding map drawer, eliminating memory recall.

💫 Tactile Animations: Switches use `AnimatedSwitcher` for smooth crossfades. Tapping floating buttons scales them down (`0.96`) for tactile feedback.

📈 Gamified Reputation & Trust

Crowdsourcing relies on user honesty. Bedrock implements an active user rating system:

  • **Reputation Tiers:** Rookie, Trusted, Expert, Veteran status paths.
  • **Calculated Scores:** Trust Coefficient and Verification Rates are updated dynamically.
  • **Social Verification:** Safe transactions block double-votes on warnings.
Fosters community-driven data accuracy

📦 Layer Decoupling

Decoupled using Domain interfaces. By separating wire-format Data Transfer Objects (DTOs) from our Domain Models, database modifications do not break the frontend.

💾 State Preservation

To prevent screen resets and conserve map zoom positions during tab navigation changes, the core scaffold body is wrapped in an `IndexedStack` widget.

🔐 Secure Storage Auth

Social sign-ins use the Google SDK. Biometrics credential tokens are encrypted in hardware-backed storage via `flutter_secure_storage`.

System Improvements

Debugging Key Usability Blockers

10 / 11

1. Silent Exception Swallowing (Diagnostics)

Data sources swallowed exceptions silently. If queries failed due to network errors or permissions, they returned empty fallbacks with no log history, violating usability diagnostics rules.

// Old implementation:
try { ... } catch (_) { return 0; }
// New implementation:
try { ... } catch (e, stack) {
  LoggerService.logError(e, stack, context: 'Admin');
  return 0;
}

2. Owner Verification (`isOwnReport` Logic)

The system compared user display names rather than unique UIDs. If two users had identical usernames, they could edit or delete each other's reports.

🛠️ Symmetrical UID Matching

Refactored the checking logic to compare unique user identifiers:

isOwnReport: currentUserId != null &&
    dto.reporterId == currentUserId
